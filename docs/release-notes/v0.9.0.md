# Haspel v0.9.0 — 2026-05-25

## Summary

The first release you can **download and install**, with the **user guide now built
into the app** and reachable from a new **Help** menu entry.

## Added

### In-app Help / bundled user guide

The user guide (14 chapters, generated from `docs/user-guide/*.md`) is now part of the
build and bundled into the installer. A new **Help** entry in the nav opens it on a
`/help` route, embedded in the app so the navigation stays visible and there's always
a way back.

- `npm run build` now runs the docs builder (`build-docs.mjs --no-pdf`) after the Vite
  build, writing `app/dist/docs/Haspel-User-Guide.html`. Because the Tauri bundle is
  the whole of `app/dist`, the manual ships inside the installer — previously the
  installed app had no manual at all.
- The guide is embedded via a same-origin iframe, so it behaves the same in the
  browser/PWA and inside the Tauri webview (no reliance on opening a new window).
- The PWA service worker no longer serves the SPA fallback for `/docs/`
  (`workbox.navigateFallbackDenylist`), so the guide loads correctly under an active
  service worker.

## Fixed

### The Tauri installer builds on Windows (first shipped artifact)

`npm run app:build` had never worked on Windows: it invoked the CLI as
`app/node_modules/.bin/tauri` — a forward-slash path with no `.cmd` extension that
`cmd.exe` can't resolve (`'app' is not recognized`). It now calls bare `tauri` (npm
puts `node_modules/.bin` on `PATH`, so it resolves cross-platform even after `cd ..`).
The stale `0.4.0` version in `app/package.json`, `src-tauri/tauri.conf.json`, and
`src-tauri/Cargo.toml` was synced to the real version so the bundle is named correctly.

Produces:

- `Haspel_0.9.0_x64-setup.exe` — per-user (NSIS), no admin required. **Recommended.**
- `Haspel_0.9.0_x64_en-US.msi` — per-machine (requires admin).

## Known limitations

- **Unsigned installer.** Windows SmartScreen will warn ("Windows protected your PC");
  choose **More info → Run anyway**. Code-signing is still on the roadmap.
- **Auto-update and macOS / Linux builds** are not done yet — the code is
  cross-platform, packaging is the remaining work.

## Deployment notes

- No data migration. Per-install data stays at `%APPDATA%\Haspel\data`, shared with the
  dev/PWA build.
- Build the installer with `npm run app:build` from `app/` (sidecar + bundle), or
  rebuild just the docs bundle with `scripts/build-prod.ps1 -DocsOnly`.
