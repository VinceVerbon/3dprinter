# Haspel v0.4.0 — 2026-05-24

## Summary

The release that makes Haspel a **real, installable desktop app**. Alongside the
Tauri packaging, v0.4.0 folds in a large batch of work: a configurable AI
provider, per-install user data, filament history, a packaging split, full
end-user documentation, and a clean fresh-user test harness.

## Highlights

- **Installable native app via Tauri 2.** Windows `.msi` + `.exe`: a small
  Rust/WebView2 shell that bundles the Vue frontend and runs the Node helper as a
  **sidecar** (bundled to a standalone exe via esbuild + pkg). The read-only
  catalog/demo seed ships as app resources; user data stays at
  `%APPDATA%\Haspel\data`, so an installed Haspel sees the same inventory as the
  dev build. Build with `npm run app:build`, dev with `npm run app:dev`.
- **Configurable AI provider.** Choose Claude CLI (default — your own Pro/Max
  quota, no key), Anthropic / OpenAI / Gemini / OpenRouter API keys, or none
  (manual entry). Per-task model selection + a Test-connection self-test.
- **Per-install user data outside the repo.** OS-conventional dirs
  (`%APPDATA%\Haspel\data`, `~/Library/Application Support/Haspel`,
  `$XDG_DATA_HOME/haspel`), overridable via `HASPEL_DATA_DIR`; one-time migration
  from the old in-repo location. Repo seeds reset to empty; "Load demo data"
  populates a fresh install.
- **Filament history / archive.** Removing a filament archives the full record
  (per-install, newest-first) instead of discarding it; restore or hard-delete
  from a History modal.
- **On-spool vs refill packaging split.** Filament inventory tracks `on_spool` +
  `refill` independently of the sealed/open/in-use breakdown, with a live balance
  check. Titles are stored clean (no "refill"/weight/diameter cruft).
- **End-user documentation + production doc bundle.** A 13-chapter user guide
  builds into a single HTML + PDF.
- **Fresh-user test harness (`stest` / `dtest`).** Spin up a fully isolated
  second instance (its own ports + throwaway data) to test exactly what a
  downloaded user gets, then destroy it.
- **Haspel branding + labels.** "H-in-a-spool" identity across icons/manifest;
  configurable label-sheet presets (Avery Zweckform 6138 + Decadry), brand-logo
  manager, AMS-aware labels, server-side PDF rendering.

## Deprecated

- The PowerShell launcher (`scripts/start.ps1`) and desktop-shortcut installer
  remain for the dev workflow but are superseded by the Tauri installer for
  end users.

## Known issues

- **One helper port.** The installed app and a dev/PWA instance both bind
  `127.0.0.1:5174` — don't run both at once.
- **Label PDF uses Edge headless** on the helper side; the cross-platform
  `window.print()` path is future work.
- **API-provider swatch** runs from model knowledge only (no live web); only the
  Claude CLI path fetches live colours.
- Per-supplier swatch resolvers beyond Bambu still use the generic AI fallback.

## Deployment notes

- Install the `.msi` (per-machine) or `-setup.exe` (per-user, no admin) from
  `src-tauri/target/release/bundle/`. Data persists across reinstalls.
- Not yet included: code-signing (SmartScreen will warn), auto-updater, a CI
  release pipeline, and macOS/Linux builds.
