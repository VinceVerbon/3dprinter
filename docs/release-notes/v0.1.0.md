# 3dprinter v0.1.0 — first cut

**Date**: 2026-05-08

## Summary

The first usable build of the personal Bambu Lab P2S supplies database. End-to-end flow works: click a taskbar shortcut → the app opens in an Edge app-window → add a filament with manual swatch entry (incl. multicolor gradients up to 5 stops) and 1–5 star rating → save → close the window → the helper process self-cleans within ~45 s. AI-driven filament enrichment (P2S+AMS 2 Pro compatibility, drying spec, print/bed temps, abrasive flag) is wired up via the `claude` CLI using the user's Pro/Max subscription — no API key required.

This release was scaffolded across three parallel sessions ("Chats A/B/C") coordinating via `crosslog.md` and isolated git worktrees, with file-boundary specs in `docs/parallel-work.md`. Worth highlighting: the multi-chat workflow worked — Chat A built the helper service while Chat B was still wiring the frontend; Chat C joined late to populate the inventory. Zero merge conflicts.

## Highlights

### Filament inventory (manual entry)
- Brand, exact product name, variant/color name, plus optional supplier-stable identifiers for future automated resolution: `sku`, `product_url`, `color_code`, `rfid_uid` (Bambu AMS), `ean`.
- Swatch UI: color picker with 1–5 multicolor stops rendered as a gradient. Effects pickers for matte, silk, sparkle, marble, metallic, glow, multicolor, translucent.
- 1–5 star rating, spool state (sealed / open / in-use / empty), free-text notes, optional purchase metadata (date, price, source, order ref).
- List with edit/remove. Empty state. Save button persists to `data/filaments.json` via the helper; localStorage fallback when helper offline.

### AI usage lookup ("Lookup AI" button)
- Posts brand + name to `POST /api/lookup-filament` on the helper.
- Helper spawns `claude --print --output-format json --model claude-sonnet-4-6` with the prompt piped via stdin (avoids cmd.exe escape issues with `|` chars in the JSON schema).
- Returns: type (PLA/PETG/PA-CF/…), abrasive flag, P2S+AMS 2 Pro compat, drying temp+hours+desiccant, print/bed temp ranges, usage notes, annealable flag.
- Cached per `brand|name` in `data/ai-cache.json` so re-opens are instant.
- Uses your Claude subscription; no Anthropic API key.

### Helper service
- Single-file Node ESM at `helper/index.mjs`, no runtime deps, 294 LOC.
- Endpoints: `/healthz`, `/heartbeat`, `/data/<file>.json` (read), `/save-data` (atomic write), `/lookup-filament`, `/lookup-swatch` (stub).
- Listens on `127.0.0.1:5174` only. CORS pinned to `http://127.0.0.1:5173`.
- Path-traversal guards on read and write. Filename regex `^[a-z0-9-]+\.json$` enforced.
- Heartbeat watchdog: 5 s tick, `process.exit(0)` after 45 s without a beat. `WATCHDOG_DISABLED=1` escape hatch for debugging.

### Lifecycle (no manual intervention)
- Pin `scripts/start.ps1` (or its desktop shortcut from `install-shortcut.ps1`) to the taskbar.
- One click → Vite dev server starts → Vite plugin spawns the helper → app opens in Edge `--app` mode.
- Browser tab emits a heartbeat to the helper every 15 s. Closing the tab stops the heartbeat → the helper exits within 45 s.
- No tray icon, no zombie processes, no manual stop ritual.

### Smaller features and fixes
- Empty-spool counter with byType breakdown.
- Settings page with default brand and AI model selector (informational; helper currently pinned to Sonnet 4.6).
- Multi-chat coordination protocol documented in `crosslog.md`. Worktree convention at `worktrees/3dprinter\<branch>` with `scripts\new-worktree.ps1` helper.

## Known issues

- **Vite dev server stays up after closing the PWA window.** Only the helper has the heartbeat watchdog; Vite continues until you stop it manually (Task Manager → end the `node.exe` on port 5173, or close its hidden console). Wiring Vite to the watchdog is v0.2 work.
- **Catalog seed missing.** `data/catalog/replacement-parts.json` and `data/catalog/consumables.json` were not authored in v0.1. The shopping-list and replacement-parts UX depend on these and are deferred until they land.
- **Per-supplier swatch resolvers stubbed.** `POST /lookup-swatch` always returns `{ hex: null, source: null, confidence: "none" }`. Bambu Filament Library + SUNLU + 123-3d.nl + RealFilament + generic fallback land in v0.2.
- **Accessory CRUD UI scaffolded but not implemented.** Add/list/edit lands in v0.2 — the data model and persistence layer already support it.
- **Shopping list page scaffolded but not implemented.** v0.2.
- **`claude` CLI must be on PATH and authenticated.** If `claude` isn't installed or the session expired, the Lookup AI button surfaces the error in the form but doesn't auto-recover.

## Deployment notes

This is a local-only Windows app — no deployment infra. To set up on a fresh Windows workstation (local or laptop):

1. Install Node 22+, npm, Microsoft Edge, and the Anthropic Claude Code CLI.
2. Clone via `git clone https://github.com/VinceVerbon/3dprinter.git <repo-dir>`.
3. `cd app && npm install`.
4. `.\scripts\install-shortcut.ps1` to create the desktop shortcut.
5. Right-click the shortcut → Pin to taskbar.

`docs/install.md` has the full walkthrough.
