# Haspel v0.6.1 — 2026-05-24

Hotfix for the v0.6.0 brand-store feature.

## Fixed

- **"Update store" did nothing but error ("fetch failed: not found").** The
  helper exposed the fetch endpoint as `/api/fetch-store`, but the dev/preview
  proxy strips the `/api` prefix before forwarding to the helper (every other
  helper route is registered without it). So the real request arrived as
  `/fetch-store`, matched no route, and returned 404. Renamed the route to
  `/fetch-store`. The v0.6.0 brand-store fetch was effectively dead through the
  UI; this restores it. Verified end-to-end: the proxied call returns the full
  store list (72 items, ~34 with EUR prices) and the "Add from store list"
  picker renders them.
- **A corrupt `store-lists.json` no longer blanks the Shopping page.** The store
  loader now coerces a non-array payload to an empty list instead of letting the
  page crash.

## Note

If you tested v0.6.0 and saw "fetch failed: not found", it was this routing bug —
not your data or setup.
