# Haspel v0.8.0 — 2026-05-25

## Summary

A stability fix plus two convenience features: navigation that recovers itself
after an app update, a one-click way to fill in all missing brand logos, and an
edit form that actually scrolls into view when you start editing.

## Fixed

### Navigation silently died after an app update

The most important fix. Each page in Haspel is loaded on demand (code-split), and
the app is a PWA that updates itself in the background. When a new build's service
worker activated, it cleaned out the previous build's cached page chunks — so a
window still running the old version would ask for a page file that no longer
existed, the load would fail, and the navigation would be **abandoned with no
error**. The visible symptom: a nav link (or the first-run **Add a printer**
button) that did nothing, dumping you back where you started.

The router now detects that specific failure and **reloads to the page you were
trying to reach** — served by the freshly-activated service worker, i.e. a
consistent build — with a guard against reload loops. Navigation heals itself
instead of dying.

Proven by forcing the exact failure (service worker disabled, a page chunk
removed) and confirming the app recovers onto the target page; confirmed it never
triggers a spurious reload during normal use.

## Added

### Brand logos — "Fetch all missing"

The **Brand logos** page (under Labels) gains a **Fetch all missing** button. It
auto-fetches a logo for every brand that doesn't have one yet, or whose previous
auto-fetch came up empty. It runs several in parallel, shows live progress
(`Fetching 3/12… (2 found)`), and saves once at the end. Brands you've explicitly
set to **Text only** are left alone. The button's count shows how many are still
outstanding, so you can run it again later for any that couldn't be found.

## Changed

### Editing a filament scrolls to the form

The add/edit form lives at the top of the Filaments page. Editing a card further
down used to leave the viewport on the card, so it wasn't obvious you'd entered
edit mode. Selecting **Edit** (or **Add filament**) now smooth-scrolls the form
into view.

## Deployment notes

- No data migration. Per-install data (`%APPDATA%\Haspel\data`) is untouched.
- Existing installs pick up the navigation self-heal on their next update; a
  window stuck on a pre-update build just needs one reload to load v0.8.0.
