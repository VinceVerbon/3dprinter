# 3dprinter v0.2.0 — catalogue, lifecycle, and the AI-fill loop

**Date**: 2026-05-08

## Summary

The first build that's actually useful day-to-day. v0.1 had the scaffold and a working manual entry path; v0.2 turns it into a real inventory: 22 catalogued filament records covering 26 physical spools, every record AI-enriched (P2S compatibility, drying spec, print/bed temps), accessories and a shopping list with a catalog picker, and a PDF order-import flow. The lifecycle is finally complete — closing the PWA window now tears down both the helper *and* Vite (the v0.1 known issue is fixed).

This release was scaffolded across three parallel sessions ("Chats A/B/C") coordinating via `crosslog.md` and isolated git worktrees, with file-boundary specs in `docs/parallel-work.md`. Three simultaneous chats, ~15 commits, zero merge conflicts in `data/` or `app/src/` (only in `app/vite.config.ts` once, when both Chat A and Chat B were rebuilding it from the bare scaffold). The architecture is now documented in `docs/architecture.md`.

## Highlights

### Inventory is real now

22 records / 26 spools captured into `data/filaments.json`:

| Brand | Records | Spools |
|---|---|---|
| Bambu Lab | 14 | 19 |
| 123-3D | 3 | 3 |
| Real Filament | 4 | 4 |

Every record has its `ai` block populated by the batch lookup pass. Bambu records carry full SKUs (e.g. `A00-K0-1.75-1000-SPL`), color codes, and `product_url` to `eu.store.bambulab.com`. Real Filament records reference 123-3d.nl (their NL distributor).

### Filament data model — one record per SKU

Replaced the v0.1 per-spool model with per-record `inventory: { sealed, open, in_use }`. Each record describes one (brand, name, variant); multiple physical spools are counts, not duplicates. `FilamentForm.vue` swapped the state dropdown for three count inputs + a spool-weight (g) field. `FilamentCard.vue` shows `N spools (a sealed, b open, c in use)`.

### Lifecycle complete (Chunk D)

The v0.1 known issue — Vite kept running after closing the PWA window — is fixed. The Vite plugin now spawns the helper with `VITE_PID=<vite-pid>` in env and registers shutdown hooks on `SIGINT` / `SIGTERM` / `process exit` / `httpServer close` to kill the helper child. The helper, on watchdog timeout (`> 45 s` without a heartbeat), kills the Vite parent before exiting.

Closing the Edge window stops heartbeats → 45 s later helper kills Vite, Vite kills helper, both processes are gone. Either side dying first cleans up the other. Standalone helper (no `VITE_PID`) is unaffected.

### Filament detail modal + filters + batch AI fill

- **Click any filament card** → modal showing SKU, color code, EAN, RFID UID, product URL, full P2S+AMS 2 Pro AI block, drying spec, print/bed temps, annealability, purchase metadata, and an inventory tile breakdown. Re-lookup button refreshes AI inline and persists to disk.
- **Filters**: type (PLA / PETG / ABS / TPU / …), effect (matte / silk / sparkle / metallic / glow / multicolor / translucent / transparent), and color family (HSL bucketing into 12 visible buckets — red / orange / yellow / green / cyan / blue / purple / pink / brown / black / gray / white). Multicolor filaments match if *any* stop falls in the selected family. Only families present in current data show as buttons.
- **Batch "Fill missing AI" button** runs `/api/lookup-filament` sequentially for every record without an `ai` block, with live progress (`Filling AI… 2/5`). Persists once at the end. Hidden when nothing's missing.
- **Print + bed temp chips** on every card (Thermometer + Square icons). Hidden when no AI data, so cards stay clean for un-looked-up entries.

### Accessories + shopping list

- **Accessory CRUD** (nozzles, hotends, build plates, AMS parts, fans, belts, lubricants, glues, desiccants, tools, cleaning) with category filter.
- **Shopping list page**: free-text entry + "Add from catalog" picker pulling from `data/catalog/replacement-parts.json` (33 P2S parts) and `data/catalog/consumables.json` (16 entries biased to NL retailers — Magigoo, 3DLAC, silica, Sunlu FilaDryer, Capricorn PTFE, IPA, brass brush). Tick-off state persists. Print-friendly view.

### PDF order import (frontend done, helper queued)

Drag a Bambu Lab EU / 123-3d.nl / Amazon NL invoice PDF onto the Filaments page. Frontend validates, posts to `/api/import-order`, shows a preview/confirm UI with auto-matching by SKU/EAN/brand+name+variant. Matches bump `inventory.sealed`; misses create new entries with full purchase metadata.

The helper-side `/import-order` endpoint is queued as Chunk F for v0.3 — until then the upload returns 404 and the UI shows the error. The frontend is shipping now so it's ready to light up the moment the helper endpoint lands.

### UX polish

- **RatingStars** overhaul: brighter unselected stroke, hover preview, scale-on-hover, optional readonly mode (used in detail header), explicit clear button. Old version looked disabled against the dark background — multiple users would've thought rating was broken.
- **`OrderImportReview.vue`** now handles accessories + consumables, not just filaments.

## Deprecated / removed

- `Filament.spool_state` (per-spool enum) — removed in favour of `Filament.inventory: { sealed, open, in_use }` counts. Pre-v0.2 records would need migration; this only matters for fresh-install users who never had real data.

## Known issues

- **Per-supplier swatch resolvers stubbed.** `POST /lookup-swatch` always returns `{ hex: null, source: null, confidence: "none" }`. Bambu Filament Library + SUNLU + 123-3d.nl + RealFilament + Firecrawl fallback land in v0.3 (Chunk E).
- **Helper `/import-order` endpoint not implemented.** PDF drop-zone calls `/api/import-order` but the helper currently 404s. Chunk F for v0.3.
- **Bambu PLA Basic Green/Gray defaults assumed** — Chat C picked G6/D0 for the catalogue; user should confirm against actual labels (could be Bright Green G3 / Mistletoe Green G2 / Silver D1 / Dark Gray D3).
- **Real Filament PETG Pink/Orange `product_url`** points to category page, not exact SKU page — Chunk E swatch resolver will re-fetch.
- **TPU for AMS color codes** are 5-digit MakerWorld codes (53101/53600/53500), not the 2-char SKU codes used elsewhere — Chunk E resolver needs to handle both formats.
- **`claude` CLI must be on PATH and authenticated.** If `claude` isn't installed or the session expired, lookup buttons surface the error in the UI but don't auto-recover.

## Deployment notes

Local-only Windows app — no deployment infra. To set up on a fresh workstation:

1. Install Node 22+, npm, Microsoft Edge, and the Anthropic Claude Code CLI.
2. Clone via `git clone https://github.com/VinceVerbon/3dprinter.git <repo-dir>`.
3. `cd app && npm install`.
4. `.\scripts\install-shortcut.ps1` to create the desktop shortcut.
5. Right-click the shortcut → Pin to taskbar.

`docs/install.md` has the full walkthrough; `docs/architecture.md` covers how the pieces fit.
