## [0.3.0] — 2026-05-11

### Added
- **Shopping list now totals up.** `ShoppingItem` gained `unit_price_eur?` — snapshotted from `price_eur_estimate` at the moment a catalog part is picked, so the total reflects what you expected to pay (catalog price drift later does not retroactively change your list). `ShoppingPage.vue` renders per-line `qty × unit` in EUR, plus a footer with **Estimated total** for open items and a separate "+ €X already checked off" line for done items. Items without a price (free-text, consumables that have no price field yet) render `—` and are reported as "N items without price not counted" beside the total. `CatalogPicker.vue` propagates the price into the pick payload.
- **Catalog mirror of Bambu store P2S accessories** — `data/catalog/replacement-parts.json` grows from 33 to 62 items (Bambu PTFE Tube 4000 mm cut-to-size `FAT003` added on top of the 28-item AMS / expansion / glue / tool sweep). Adds the AMS units themselves (AMS 2 Pro standalone `SAA034`, AMS HT `SAA036`), AMS expansion hardware (Switching Adapter `SAA033`, 4-in-1 PTFE Adapter `FAZ013`, Filament Buffer P2S `SA011`), TPU Feed Assist Module (`FAC234`), External Exhaust Fan Kit (`FAF025`) + air filter (`FAC159`) + updated rear panel (`FAS071`), Spool Holder P2S (`FAC183`), Floating Damper Feet (`FAZ051`), Nozzle Cleaning Sponge (`FAC323`), Bambu Reusable Spools low/high-temp (`SGL002`/`SGL003`), Bambu Glue Stick (`SGL001`) + Liquid Glue (`LGL001`), and the full Maker's Supply tool range (long-nose pliers, mag-alloy scraper, precision/flush cutter, trimming kit, ESD tweezers, needle file set, screwdriver kit, L-hex wrench, hand drill kit, cutting mat, USB-C heat-set soldering iron). Every item carries Bambu SKU, EU EUR price estimate, store URL, and a P2S-specific note. `data/catalog/consumables.json` gains Bambu OEM glue stick + 60 mL liquid glue as restockable consumables. Source: bambu-p2s-specialist research vs. eu.store.bambulab.com + 3DJake NL pricing.
- **Real swatch resolver (Chunk E, Bambu-prioritised, multicolor-aware).** `POST /api/lookup-swatch` is no longer a stub. Given `{ brand, name, variant?, sku?, color_code?, product_url? }` it returns `{ hex, stops[], effects[], source, confidence, notes }`. Bambu Lab inputs trigger a dedicated prompt that WebFetches `eu.store.bambulab.com` / `wiki.bambulab.com` + the official Bambu hex-code PDFs for exact SKU matches. Other brands fall through to a generic AI resolver. Results are cached in `data/ai-cache.json` under `swatch:<brand>|<name>|<variant>` so the same product never costs subscription quota twice; pass `force: true` to bypass. Helper invokes `claude` from `os.tmpdir()` with `--allowedTools WebFetch,Read --permission-mode bypassPermissions --no-session-persistence --append-system-prompt` framing — same hardening as `/import-order`. Multicolor filaments (PLA Silk Multi-Color, Galaxy, Marble, Dual Color) come back with all visible stops in `stops[]` and `effects` includes `'multicolor'`. Spec frozen at `docs/chunk-e-swatch-resolver.md` so the work survives session boundaries. Verified against Bambu PLA Basic Black (`#000000`, confidence=high) and PLA Silk Multi-Color "Ochtendglans" (4 stops — orange/teal/purple/pink dawn gradient, confidence=medium with retailer citations).
- **Order-import drop-zone now infers swatches inline (Tier 1) and resolves them post-import (Tier 2).** `/import-order` claude prompt extended with `hex`, `stops[]`, `effects[]` per item — the user sees a sensible colour the instant the review modal opens (cost-free, same claude call). `OrderImportReview.apply()` then fires `/api/lookup-swatch` per newly-created filament in parallel; resolved swatches overwrite the Tier 1 hint and persist via `filaments.save()`. Helper offline / claude failure → Tier 1 hint stays, never regresses to grey.
- **"Refresh swatches" button on the Filaments page.** Counts and resolves any filament still stuck on the grey `#888888` `source: 'manual'` placeholder (the legacy default for pre-Chunk-E imports). Progress reported inline (`Resolving swatches… 3/13`). Hidden when nothing's grey.
- **One-shot CLI backfill at `scripts/backfill-swatches.mjs`.** Same selection criteria as the button — `swatch.hex==='#888888' && stops==['#888888'] && source==='manual'` — but runnable headless. Pre-flight `git status --porcelain data/filaments.json` refuses to run on a dirty working copy unless `SKIP_GIT_CHECK=1` is set. Atomic write, idempotent, rollback = `git restore data/filaments.json`.
- **Multicolor filter on Filaments page.** New pseudo-family in the color-family filter row alongside Red/Orange/Yellow/etc. Matches `swatch.effects.includes('multicolor')` OR ≥2 distinct hex stops. Swatch dot uses a CSS conic-gradient through all 12 single-colour family hues. Short-circuits so a Galaxy filament shows up only under Multicolor, not also under its dominant colour.
- **Backfill executed.** All 13 grey filaments in `data/filaments.json` (Bambu TPU 85A/90A, PLA Matte refills, PLA Silk Multi-Color) replaced with resolved swatches sourced from the official Bambu hex-code PDFs. The Silk Multi-Color "Ochtendglans" canary came back with 4 stops; "Zuidstrand" with 2 stops; the remaining single-colour items with high-confidence Bambu source.

### Fixed
- **Helper JSON parsing is now lenient.** `runClaude` callers used to do `JSON.parse(stripFences(inner))` which would fail if claude prefaced its JSON with a sentence of prose (e.g. "Found it confirmed in the raw GitHub colors.json…  {…}"). Replaced with `parseLooseJson` which tries direct parse → strip-fences → extract first balanced `{…}` block. Caught a real failure during the swatch backfill (PLA Matte refill "Appelgroen") and applies to `/lookup-filament`, `/lookup-swatch`, and `/import-order` uniformly.
- **PDF drop-zone now actually imports orders.** The frontend `OrderDropZone.vue` was wired in v0.2 but the matching helper endpoint (`POST /import-order`, Chunk F) was never implemented — drops surfaced as "Failed to fetch" in the UI because the helper's 404 fallthrough never consumed the multipart body, so Node tore the connection down mid-upload. Implemented the endpoint: streams the PDF to a temp file via `busboy` (10 MB cap → 413), shells out to `claude --print --output-format json` from `tmpdir()` (so the project's `CLAUDE.md`/crosslog can't bleed into the model's context and trip its prompt-injection guard) with `--allowedTools Read --permission-mode bypassPermissions --no-session-persistence --append-system-prompt` framing the call as the legitimate extraction backend it is, then parses the wrapper, strips fences, validates shape, and returns the contract documented in `docs/parallel-work.md` §326–356. Verified end-to-end with a synthetic receipt PDF: vendor + order_ref + date + line items (incl. SKU + variant + unit price) all populated. Temp PDF is unlinked in `finally`. New dep: `busboy@^1.6` in `helper/package.json`.
- **Save button on Add/Edit forms now actually saves.** `FilamentForm.vue` and `AccessoryForm.vue` were calling `structuredClone(form)` and `structuredClone(props.initial)`, but the form is a `reactive()` Vue Proxy whose nested objects (e.g. `swatch`, `inventory`) are also Proxies — Chromium's `structuredClone` rejects those with `DataCloneError: #<Object> could not be cloned`, so the submit handler threw before `emit('submit', …)` ever ran. The form has `@submit.prevent`, so the only user-visible symptom was "the Save button does nothing." Replaced with a JSON-roundtrip clone (`JSON.parse(JSON.stringify(v))`) since both records are pure JSON. Affects every Add/Edit path on the Filaments and Accessories pages, including the edit launched from the filament detail modal.
- **Desktop launcher (`scripts/start.ps1`)** now actually opens the app on Windows. Four bugs were stacked: (1) `Get-Command npm` returned the `npm.ps1` shim, which `Start-Process` cannot launch — now resolves `npm.cmd` first; (2) `msedge.exe` is not on PATH on this machine (Edge installs under `Program Files (x86)`), so the Edge branch was skipped — added a path-probe fallback to the standard install locations for both Edge and Chrome; (3) the script was UTF-8 without BOM, so `powershell.exe` 5.1 read the em dashes as Windows-1252 and parse-failed silently in the hidden window — re-saved with UTF-8 BOM; (4) **root cause of the user-visible failure**: `app/vite.config.ts` had `server.host: 'localhost'`, which on Windows resolves to `::1` first, leaving Vite IPv6-only — so the `127.0.0.1:5173` health probe and `msedge --app=http://127.0.0.1:5173/` both got connection-refused. Pinned Vite to `127.0.0.1` to match the helper.

### Known issues
- **Per-supplier swatch resolvers beyond Bambu remain on the AI-generic fallback.** 123-3d.nl / RealFilament / SUNLU-specific scrapers were explicitly scoped out of Chunk E (see `docs/chunk-e-swatch-resolver.md` §"Out of scope"). Other-brand lookups still resolve via claude's training knowledge + optional WebFetch — good enough in practice but lower confidence than Bambu.
- **`/api/lookup-swatch` requires `claude` CLI auth.** Same as `/lookup-filament` — if the local claude session is expired, the resolver fails open (keeps Tier 1 hint on import; "Refresh swatches" reports failures inline).
- **Backfill script (`scripts/backfill-swatches.mjs`) refuses to run on a dirty `data/filaments.json`** without `SKIP_GIT_CHECK=1`. By design — protects against accidental overwrite of in-flight inventory edits.

## [0.2.0] — 2026-05-08

### Added
- **Vite ↔ helper lifecycle wiring (Chunk D)**: `helperPlugin` in `app/vite.config.ts` now spawns the helper as a child process with `VITE_PID=<vite-pid>` in its env, and proxies `/api/*` (rewriting `/api` → `/`) and `/data/*` to the helper on `127.0.0.1:5174`. The helper, on watchdog timeout / SIGINT / SIGTERM, sends `SIGTERM` to that PID before exiting. Closing the PWA window now tears down both processes. Standalone helper (no Vite parent) is unaffected.
- **Filament catalogue seed** at `data/filaments.json` — 22 records covering 26 physical spools (19 Bambu Lab, 3 123-3D, 4 Real Filament). Bambu records carry full SKUs (e.g. `A00-K0-1.75-1000-SPL`), color codes, and `product_url` to `eu.store.bambulab.com`. Real Filament records reference 123-3d.nl product/category pages. Every record's `ai` block is populated (P2S compatibility, drying, print/bed temps, usage notes, abrasive flag).
- **Catalog seed**: `data/catalog/replacement-parts.json` (33 P2S parts with verified Bambu SKUs and EUR price estimates) + `data/catalog/consumables.json` (16 entries biased to NL retailers — Magigoo, 3DLAC, silica, Sunlu FilaDryer, Capricorn PTFE, IPA, brass brush).
- **Accessory CRUD UI**: list / add / edit / remove with category filter (nozzles, hotends, build plates, AMS parts, fans, belts, lubricants, glues, desiccants, tools, cleaning).
- **Shopping list page**: free-text entry + "Add from catalog" picker pulling from the catalog seeds. Tick-off state persists. Print-friendly view.
- **PDF order import**: drag-drop a Bambu / 123-3d.nl / Amazon NL invoice PDF onto the Filaments page → preview + confirm UI → match on SKU/EAN/brand+name+variant. Matches bump `inventory.sealed`; misses create new entries with full purchase metadata. Frontend complete; helper-side `/import-order` endpoint queued as Chunk F for v0.3.
- **Filament detail modal** (click any card): SKU, color code, EAN, RFID UID, product URL, full P2S+AMS 2 Pro AI block, drying spec, print/bed temps, annealability, purchase metadata, inventory tiles. Re-lookup button refreshes AI inline and persists to disk.
- **Filters on filaments page**: by type (PLA/PETG/ABS/…), by effect (matte/silk/sparkle/…), and by color family (HSL bucketing into 12 visible buckets — red/orange/yellow/green/cyan/blue/purple/pink/brown/black/gray/white). Only families present in current data show as filter buttons. Multicolor filaments match if any stop falls in the selected family.
- **Batch "Fill missing AI"** button on filaments page: sequentially runs `/api/lookup-filament` for every record without an `ai` block. Live progress (`Filling AI… 2/5`), persists once at the end, counts failures separately so a single bad lookup doesn't abort the batch. Hidden when nothing is missing.
- **Print + bed temp chips** on filament cards (Thermometer + Square icons with the AI temp ranges). Hidden when no AI data, so cards stay clean for un-looked-up entries.
- **`Effect` enum gained `transparent`** (in addition to `translucent`) for fully see-through filaments.
- **Architecture documentation** at `docs/architecture.md` covering process model, lifecycle, helper service, AI-lookup chain, persistence flow, color bucketing, multi-chat workflow, and file map.

### Changed
- **Filament data model**: replaced per-spool `spool_state` with per-record `inventory: { sealed, open, in_use }`. Each record now describes one (brand, name, variant) — i.e. one SKU-equivalent — and physical spools of that SKU are counted, not duplicated as separate records.
- `FilamentForm.vue`: state dropdown replaced with three count inputs + a spool-weight (g) field.
- `FilamentCard.vue`: now shows `N spools (a sealed, b open, c in use)`, plus print + bed temp chips when AI data is present.
- `RatingStars.vue`: brighter unselected stroke, hover preview, scale-on-hover, optional readonly mode (used in detail header), explicit clear button. Addresses confusion where stars looked disabled.
- `OrderImportReview.vue`: now handles accessories + consumables as well as filaments.

### Fixed
- v0.1 known issue: closing the PWA `--app` window left `vite` running on port 5173 because only the helper self-terminated via the heartbeat watchdog. Now resolved via Chunk D.
- Detail-modal `Re-lookup` button used to update only a local ref and never write back to disk. Now pushes the result to the filaments store and triggers `save()`.
- `vite.config.ts` was at one point reduced to the bare scaffold by a tooling glitch, breaking Tailwind, the helper plugin, and the PWA wiring. Rebuilt and merged with Chunk D's lifecycle hooks.

### Known issues
- **Per-supplier swatch resolvers stubbed.** `POST /lookup-swatch` always returns `{ hex: null, source: null, confidence: "none" }`. Bambu Filament Library + SUNLU + 123-3d.nl + RealFilament + Firecrawl fallback land in v0.3 (Chunk E).
- **Helper `/import-order` endpoint not implemented.** PDF drop-zone calls `/api/import-order` but the helper currently 404s. Chunk F for v0.3.
- **Bambu PLA Basic Green/Gray defaults assumed** — Chat C picked G6/D0; user should confirm against actual labels.
- **Real Filament PETG Pink/Orange `product_url`** points to category page, not exact SKU. Chunk E swatch resolver will re-fetch.
- **TPU for AMS color codes** are 5-digit MakerWorld codes (53101/53600/53500), not the 2-char SKU codes used elsewhere — Chunk E resolver needs to handle both formats.
- **`claude` CLI must be on PATH and authenticated.** If `claude` isn't installed or the session expired, lookup buttons surface the error in the UI but don't auto-recover.

## [0.1.0] — 2026-05-08

### Added
- Initial scaffold: `.gitignore`, `VERSION`, `README.md`, `CLAUDE.md`, `crosslog.md`, `CHANGELOG.md`, `docs/releaselog.md`, `docs/release-notes/`.
- Worktree convention: `worktrees/3dprinter\<branch>` with helper `scripts\new-worktree.ps1`.
- Registered in `~/.claude/project-ledger.json` under `systems.local`.
- GitHub remote: `vinceverbon/3dprinter` (private).
- **Vue 3 frontend (`app/`)**: Vite + TypeScript + Tailwind CSS 4 + Pinia + Vue Router (hash mode) + Lucide icons + `vite-plugin-pwa` for installable PWA. Custom `helperPlugin` in `vite.config.ts` spawns the helper process alongside `npm run dev` and tears it down on Vite shutdown. App emits a heartbeat to `/api/heartbeat` every 15 s.
- **Filament CRUD**: list / add / edit / remove with manual swatch entry — color picker, 1–5 stop multicolor gradient, effects (matte / silk / sparkle / marble / metallic / glow / multicolor / translucent), 1–5 star rating, spool state, notes. Supplier-stable identifiers (sku, product_url, color_code, rfid_uid, ean, purchased) on the data model for future automated lookups.
- **AI usage lookup**: "Lookup AI" button on filament form posts brand+name to `/api/lookup-filament`; renders P2S+AMS 2 Pro compatibility, drying spec, abrasive flag, print/bed temps, usage notes. Cached in `data/ai-cache.json` keyed by `brand|name`.
- **Empty-spool counter** with byType breakdown.
- **Settings page** with default brand and AI model selector.
- **Pinia stores + persistence layer**: `loadData` / `saveData` against helper `/data` + `/api/save-data`, with localStorage offline fallback.
- **Helper service** (`helper/index.mjs`, ~294 LOC, Node ESM, no runtime deps). Endpoints: `GET /healthz`, `POST /heartbeat`, `GET /data/<file>.json` (incl. one optional subdir level for catalog), `POST /save-data` (atomic tmp+rename, pretty-printed), `POST /lookup-filament` (spawns `claude --print --output-format json` with prompt piped via stdin), `POST /lookup-swatch` (stub for v0.2). Path-traversal guard, CORS pinned to dev-server origin. Heartbeat watchdog: `process.exit(0)` after 45 s without a beat.
- **Launcher** (`scripts/start.ps1`): one-click entry point that starts `npm run dev` (which auto-spawns the helper via `helperPlugin`), waits for both ports, opens the PWA in Edge `--app` mode. No-op if already running. Fallback to Chrome / default browser.
- **Desktop-shortcut installer** (`scripts/install-shortcut.ps1`): creates a `.lnk` on the Desktop pointing at `start.ps1` with the project icon. Pin manually to taskbar.
- **Install guide** (`docs/install.md`): ~2-minute setup walkthrough + troubleshooting.
- **Parallel-work spec** (`docs/parallel-work.md`): three claimable chunks (Helper / Catalog seed / Launcher) with file boundaries, API contracts, and acceptance criteria so multiple sessions can work the project simultaneously.

### Known issues
- The Vite dev server keeps running after the PWA window is closed (only the helper has a heartbeat watchdog). Stop it via Task Manager or close from its hidden console. v0.2 will wire Vite to the watchdog.
- Catalog seed (`data/catalog/replacement-parts.json`, `data/catalog/consumables.json`) was not authored in v0.1. Shopping list and replacement-parts UX wait on this.
- Per-supplier swatch resolvers (Bambu / SUNLU / 123-3d.nl / RealFilament / generic) are stubbed in the helper. v0.2 work.
- Accessory CRUD UI is scaffolded but not implemented. v0.2.
- Shopping list page is scaffolded but not implemented. v0.2.
